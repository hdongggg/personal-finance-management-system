\section{Introduction}

\subsection{Project Background}
Personal finance management is an important problem for individual users because income, expenses, and account balances change continuously over time. A database-driven system helps store financial transactions in a structured way and supports reliable summaries and reports.

\subsection{Project Objective}
The objective of this project is to build a Personal Finance Management System that allows users to manage income, expenses, bank accounts, and spending categories, while also supporting summaries, reports, and budget monitoring.

\subsection{Scope of the System}
The system focuses on the following core features:
\begin{itemize}
    \item user profile management,
    \item income and expense entry,
    \item expense categorization,
    \item bank account tracking,
    \item monthly and category-based reporting,
    \item budget checking and financial overview.
\end{itemize}

\section{System Analysis and Requirements}

\subsection{Functional Requirements}
The system shall:
\begin{itemize}
    \item store user information,
    \item record income transactions,
    \item record expense transactions,
    \item classify expenses by category,
    \item track balances of bank accounts,
    \item provide daily, monthly, and yearly summaries,
    \item support spending analysis and budget alerts.
\end{itemize}

\subsection{Non-functional Requirements}
The project uses MySQL as the database management system and Python as the application language. The system should ensure data consistency, support reusable SQL objects, and provide efficient reporting queries.

\subsection{Main Entities}
The main entities in the system are Users, ExpenseCategories, BankAccounts, Income, and Expenses. These entities are later converted from the ER design into a relational schema with primary keys, foreign keys, and necessary constraints.
