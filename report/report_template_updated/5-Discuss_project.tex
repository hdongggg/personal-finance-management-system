\section{Database Security and Administration}

\subsection{Authentication and Access Control}
For database security, the application should use a dedicated MySQL account instead of the root account. Only the required privileges such as \texttt{SELECT}, \texttt{INSERT}, \texttt{UPDATE}, \texttt{DELETE}, and \texttt{EXECUTE} should be granted to the application user.

\subsection{Backup and Recovery}
To ensure reliability, backup and recovery procedures should be prepared. A full backup of the database can be created regularly by using MySQL tools such as \texttt{mysqldump}. The backup file can later be used to restore the system if accidental data loss occurs.

\subsection{Performance Optimization}
Query performance is improved through indexing and by using views for repeated reporting tasks. This reduces the complexity of application queries and improves maintainability.

\section{Discussion}
The project demonstrates how database design, advanced SQL objects, and a simple Python application can be integrated into a practical personal finance system. One limitation is that the current implementation is still a basic version and can be further improved with a graphical interface, stronger authentication, and more advanced analytics.
